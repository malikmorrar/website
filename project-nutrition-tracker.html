Overview

The Problem

UCLA dining halls post nutrition information online, but it's inconsistent, hard to navigate, and scattered across multiple pages. Students trying to track their macros or calories have no easy way to log meals by dining hall, and the posted data doesn't always match what they'd expect from standard nutrition databases.

The Solution

I built an automated pipeline that scrapes UCLA dining menus daily using Python, parses the nutrition fields into a cleaned CSV dataset, and cross-references them against the USDA FoodData Central API to normalize macros and flag discrepancies between what UCLA posts and standard nutritional data. On top of this data layer, I built a JavaScript web app backed by Firebase where students can log their meals, view calorie totals by dining hall, and track nutrition trends over time. Versioned QA logs ensure the calorie estimates improve with each iteration.

How It Works
01
Daily Menu Scraping
Python scraper pulls UCLA dining hall menus each day, parses nutrition fields, and stores results in a versioned, cleaned CSV dataset per dining hall
02
USDA Data Normalization
USDA FoodData Central API is used to normalize macro data and flag gaps or discrepancies between UCLA's posted nutrition info and standard nutritional values
03
Firebase Data Storage
Cleaned, normalized meal data is stored in Firebase, enabling real-time reads and writes for the web app — with versioned QA logs for iterative accuracy improvements
04
Web App & Dashboards
JavaScript frontend lets students log meals, track daily calorie intake by dining hall, and view visualizations of their nutrition trends over time
